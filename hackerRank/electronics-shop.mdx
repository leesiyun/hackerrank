# Electronics Shop

🔗 [problem](https://www.hackerrank.com/challenges/electronics-shop/problem?isFullScreen=true)

주어진 예산으로 구입할 수 있는 가장 비싼 컴퓨터 키보드와 USB 드라이브를 결정하려고 합니다. 키보드와 USB 드라이브에 대한 가격표와 예산이 주어지면 구입 비용을 확인하십시오. 만약 두 가지 물건을 모두 살 수 없다면, -1을 반환합니다..

<br />

## 예제

```
b = 60
keyboards = [40, 50, 60]
drives = [5, 8, 12]
```

a는 `40 키보드와 12 USE 드라이브 = 52` 또는 `50 키보드 + 8 USB 드라이브 = 58`을 살 수 있다. 더 비싼 옵션으로 선택하고 58을 반환합니다.

<br />

## 기능 설명

아래 편집기에서 getMoneySpent 기능을 완료하십시오.

getMoneySpent에는 다음 매개 변수가 있습니다:

- int keyboards[n]: 키보드 가격
- int drives[m]: 드라이브 가격
- int b: 예산

<br />

## Return

- int: 사용할 수 있는 최대 한도 또는 두 항목 모두를 구입할 수 없는 경우 `-1`

<br />

## 입력 형식

첫 번째 줄에는 공백으로 구분된 세 개의 정수 `b`, `n`, `m` 순서로 예산, 키보드 모델의 수 및 USB 드라이브 모델의 수가 포함됩니다.  
두 번째 줄에는 `n`개의 각 키보드 모델의 가격인 공백으로 구분된 정수 `keyboard[i]`가 포함되어 있습니다.  
세 번째 줄에는 `m`개의 USB 드라이브의 가격인 공백으로 구분된 정수`drives`가 포함되어 있습니다.

<br />

## 제약

- `1 ≤ n, m ≤ 1000`
- `1 ≤ d ≤ 10^6`
- 각 품목의 가격은 `[1, 10^6]`에 포함된 범위에 있습니다.
- <br />

## 샘플 입력 0

```
10 2 3
3 1
5 2 8
```

<br />

## 샘플 출력 0

```
9
```

<br />

## 설명 0

`2nd` 키보드와 `3rd` USB 드라이브를 총 비용으로 `8 + 1 = 9`로 구매합니다,

<br />

## 샘플 입력 1

```
5 1 1
4
5
```

<br />

## 샘플 출력 1

```
-1
```

<br />

## 설명 1

`4 + 5 > 5` 예산을 초과하기 때문에 키보드 하나와 USB 드라이브 하나를 사는 방법은 없습니다. 그래서 `-1` 반환합니다.

<br />

## solution

```JS
const getMoneySpent = (keyboards, drives, b)  => {
    const result = [];
    keyboards.map((keyboard) =>
      drives.map(
        (drive) => drive + keyboard <= b && result.push(drive + keyboard)
      )
    );
    return result.length > 0 ? Math.max(...result) : '-1';
}

```
